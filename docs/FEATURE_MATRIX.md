# Feature-Matrix: Light vs. Premium Version

| Feature               | Light (Open Source) | Premium (Enterprise) |
| --------------------- | :-----------------: | :------------------: |
| Routing               |          ✔️          |          ✔️           |
| Middleware            |      ✔️ (Basic)      |     ✔️ (Advanced)     |
| Controller-System     |          ✔️          |          ✔️           |
| Datenbank-Anbindung   |      ✔️ (Basic)      |    ✔️ (ORM, Adv.)     |
| Migrationen/Seeders   |          ❌          |          ✔️           |
| Templating            |      ✔️ (Basic)      |     ✔️ (Advanced)     |
| Authentifizierung     |          ❌          |          ✔️           |
| Rollen/Rechte         |          ❌          |          ✔️           |
| Security (CSRF/XSS)   |          ❌          |          ✔️           |
| CLI-Tool              |          ❌          |          ✔️           |
| Events/Queue          |          ❌          |          ✔️           |
| Caching               |          ❌          |          ✔️           |
| API (REST/GraphQL)    |          ❌          |          ✔️           |
| Monitoring/Logging    |      ✔️ (Basic)      |     ✔️ (Advanced)     |
| Microservices-Support |          ❌          |          ✔️           |
| Tests                 |      ✔️ (Basic)      |     ✔️ (Advanced)     |
| Dokumentation         |          ✔️          |     ✔️ (Premium)      |
| Erweiterbarkeit       |     ✔️ (Module)      |   ✔️ (Premium-Mod.)   |
| Community-Support     |          ✔️          |          ❌           |
| Lizenz                |     Open Source     |   Kommerziell/Dual   |

Legende:
- ✔️ = enthalten
- ❌ = nicht enthalten
- (Basic) = Grundfunktionalität
- (Advanced) = Erweiterte Funktionalität

Diese Matrix kann als Grundlage für die Kommunikation und Entwicklung beider Versionen dienen.
